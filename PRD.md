# PRD: AIQ — рейтинг ИИ-зоркости

## Что это

AIQ — виральный интерактивный тест: пользователь различает контент, созданный ИИ и человеком, и получает балл AIQ от 0 до 100. Балл выдаётся в виде шарящегося диплома. Цель — виральная активация под совместное с РБК исследование о различении ИИ-контента. Без платного продвижения.

## Кто пользователь

Маркетолог среднего и старшего звена, креативный или арт-директор, автор профессионального Telegram-канала. Возраст 25–45, активный пользователь Telegram, приходит по ссылке из материала РБК, рассылки Kokoc или поста знакомого эксперта.

## Главная цель и метрики

- 2 000–5 000 прохождений за первый месяц
- 10–15% от прошедших шерят диплом
- 15–20 из 20–25 приглашённых экспертов постят свой результат в своём канале

## Пользовательский путь

1. Пользователь приходит на лендинг по ссылке
2. Видит экспертный лидерборд (топ-25 человек индустрии с их AIQ-баллом)
3. Жмёт «Пройти тест»
4. Проходит 8 раундов: в каждом 2 варианта одного типа, размечает каждый как «ИИ» или «человек»
5. Видит экран результата: AIQ, разбивка по модальностям, профиль смещения, кнопка скачать диплом
6. Опционально: вводит имя и соглашается на публикацию в лидерборде
7. Скачивает диплом и шерит в Telegram

## Экраны и контент

### 1. Лендинг (`/`)

Содержимое:
- Hero-заголовок и подзаголовок (финальный копирайт — от копирайтера, на этапе разработки — заглушки)
- Кнопка «Пройти тест» → переход на `/test`
- Экспертный лидерборд: таблица топ-25 — место, имя, должность, AIQ
- Внизу: соавторская плашка РБК + Kokoc, ссылка на методологию

### 2. Тест (`/test`)

Состояние теста: пользователь проходит 8 раундов последовательно.

В каждом раунде показывается:
- Прогресс-бар (1/8, 2/8, ...)
- Название типа сета («Описания товара», «Картины», «Звонки», «Песни» и т.д.)
- Таймер обратного отсчёта (для текста 45 сек, для изображения 30 сек, для видео 60 сек, для звука 50 сек)
- 2 варианта рядом:
  - Текст: текстовые карточки 300–500 знаков
  - Изображение: 2 картинки
  - Видео: 2 видеоплеера (с автостартом, без звука по умолчанию)
  - Звук: 2 аудиоплеера (с кнопкой play)
- Под каждым вариантом — переключатель «ИИ / человек»
- Кнопка «Далее» — активна только когда оба варианта размечены
- Если таймер истёк — автоматический переход с текущими ответами (неразмеченные считаются неправильными)
- Возврат к прошлому раунду невозможен

### 3. Результат (`/result`)

Содержимое:
- AIQ балл крупно: «Ваш AIQ: 72»
- Процентиль: «Лучше чем 54% прошедших»
- 4 минимальных бара по модальностям с цифрами
- Профиль смещения с иконкой (параноик / доверчивый / сбалансированный)
- Двухосный титул из матрицы балл × профиль
- Превью диплома
- Кнопка «Скачать диплом» — генерирует PNG на клиенте
- Кнопка «Поделиться в Telegram» — открывает шер с превью
- Опциональное поле «Имя для лидерборда» (макс 50 символов)
- Опциональный чекбокс «Опубликовать в публичном лидерборде»
- Кнопка «Опубликовать»

**Бары по модальностям остаются только на /result. В диплом не выносим — это намеренная оптимизация под минималистичный шеринговый артефакт (сторис-формат 1080×1920).** Содержимое диплома: брендинг РБК × Kokoc в шапке, AIQ-балл крупно, титул из getTitle, плашка профиля смещения с эмодзи, дата прохождения и домен в подвале.

### 4. Лидерборд (`/leaderboard`)

Содержимое:
- Заголовок «Топ ИИ-зорких людей маркетинга»
- Фильтр: «Все / Только эксперты»
- Таблица: место, имя, должность (если эксперт), AIQ, профиль, дата прохождения
- Сортировка по AIQ убыванию
- Показываются только результаты с флагом `show_in_leaderboard=true`

### 5. Методология (`/methodology`)

Содержимое:
- Краткое описание процедуры теста
- Источники контента (какие ИИ-модели использовались, откуда человеческий контент)
- Формула AIQ
- Кто стоит за исследованием (РБК + Kokoc)
- Контакты для прессы

## Логика теста

### Состав банка контента

34 единицы 10 типов, каждая заранее размечена как ИИ или человек. Все метаданные — в `lib/content.json`. Структура — см. `content.example.json` в корне проекта.

Аудио-звонки намеренно разделены на два типа (`phone-call-a` и `phone-call-b`) — это разные сценарии разговоров (например, разные тематики или роли участников). Разделение даёт больше вариативности при отборе пар: для модальности «звук» в сессии берутся 2 пары из разных типов, чтобы пользователь слышал не два звонка одного и того же сценария.

| Тип | Кол-во | Модальность |
|---|---|---|
| marketplace | 3 | text |
| bank | 3 | text |
| ai-note | 3 | text |
| painting | 6 | image |
| landscape | 5 | image |
| cat-video | 3 | video |
| ad-video | 2 | video |
| phone-call-a | 3 | audio |
| phone-call-b | 3 | audio |
| song | 3 | audio |

### Формирование пар

При старте приложения из 34 единиц генерируются все возможные пары внутри одного типа (комбинаторика C(n, 2)):

| Тип | n | Пар |
|---|---|---|
| marketplace | 3 | 3 |
| bank | 3 | 3 |
| ai-note | 3 | 3 |
| painting | 6 | 15 |
| landscape | 5 | 10 |
| cat-video | 3 | 3 |
| ad-video | 2 | 1 |
| phone-call-a | 3 | 3 |
| phone-call-b | 3 | 3 |
| song | 3 | 3 |
| **Итого** | **34** | **47** |

Каждая пара имеет композицию: AI+AI, AI+human или human+human. Это вычисляется автоматически из поля `source` каждой единицы.

### Выбор пар для сессии

В каждой сессии — 8 раундов, по 2 на каждую модальность.

Правила выбора:
- **Текст** (3 типа, 9 пар): выбираются 2 случайные пары, желательно из разных типов
- **Изображение** (2 типа, 25 пар): по одной паре из каждого типа (1 картина + 1 пейзаж)
- **Видео** (2 типа, 4 пары): по одной паре из каждого типа (1 котики + 1 реклама)
- **Звук** (3 типа, 9 пар): выбираются 2 случайные пары, желательно из разных типов
- В одной сессии никакая пара не повторяется

### Формулы

**AIQ:**
- `correctCount` — число правильно размеченных единиц из 16 (8 раундов × 2 единицы)
- `AIQ = Math.round(correctCount / 16 × 100)`
- Неразмеченные единицы из-за тайм-аута считаются неправильными

**Модальные субскоры:**
- Для каждой из 4 модальностей: `correct в этой модальности / 4 × 100`

**Профиль смещения:**
- `falsePositive` — раз пользователь сказал «ИИ», а на самом деле человек
- `falseNegative` — раз сказал «человек», а на самом деле ИИ
- `delta = falsePositive - falseNegative`
- `delta >= 2` → `paranoid`
- `delta <= -2` → `trusting`
- иначе → `balanced`

**Процентиль:**
- При сохранении результата считаем: какой процент всех ранее сохранённых результатов имеет AIQ ниже текущего
- Если результатов меньше 50 — показываем «среди первых X прошедших» вместо процентиля

### 12 титулов диплома

Матрица 4 диапазона балла × 3 профиля смещения. Финальные тексты от копирайтера, на этапе разработки — заглушки:

| AIQ | Сбалансированный | Параноик | Доверчивый |
|---|---|---|---|
| 85+ | AI-снайпер | Видит ИИ даже там, где его нет | Точен, когда уверен |
| 65–84 | ИИ-зоркость в норме | Видит ИИ даже в котах | Замечает машинку через раз |
| 45–64 | Половину распознаёт | Постоянно ищет ИИ — и часто находит в людях | Готов принять любого |
| Ниже 45 | Уже не различает | Никому не верит | Готов поверить любому |

## Данные

### Структура `content.json`

См. `content.example.json` в корне — там пример формата. Каждая единица:
- `id` — уникальный строковый ID
- `type` — один из 10 типов выше
- `modality` — text / image / video / audio
- `source` — `'ai'` или `'human'`
- `content` — для текста: сам текст
- `url` — для медиа: путь относительно /public

### Supabase: таблица `results`

```sql
CREATE TABLE results (
  id uuid primary key default gen_random_uuid(),
  aiq integer not null check (aiq >= 0 and aiq <= 100),
  modality_text integer not null,
  modality_image integer not null,
  modality_video integer not null,
  modality_audio integer not null,
  bias_profile text not null check (bias_profile in ('paranoid', 'trusting', 'balanced')),
  display_name text,
  expert_title text,
  is_expert boolean default false,
  show_in_leaderboard boolean default false,
  created_at timestamptz default now()
);

CREATE INDEX idx_aiq ON results(aiq);
CREATE INDEX idx_show_leaderboard ON results(show_in_leaderboard, aiq) WHERE show_in_leaderboard = true;
```

Row Level Security:
- INSERT — открыт для роли `anon`
- SELECT — открыт для роли `anon`, но только записи с `show_in_leaderboard = true`

## Анти-чит (минимальный)

- Таймер на каждый раунд с автопереходом
- Невозможность вернуться к прошлому раунду
- Один проход на устройство в течение 24 часов (через localStorage flag)
- Серверная рандомизация выбора пар, чтобы у разных пользователей был разный тест

- Лидерборд показывает реальные результаты
- 20–25 экспертов смогли пройти и согласились на публикацию
